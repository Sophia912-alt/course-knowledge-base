## 这个知识库能做什么

上完一门课后，你只需要做两件事：

1. **把课件放进来**
2. **把录音转写放进来**

之后系统会自动完成这些事情：

- 🧹 **整理** — 自动清洗格式、去噪音
- 🧠 **分析** — AI 会阅读每份资料，提取出重要知识点和结论
- 📝 **写总结** — 为每节课生成一份独立的笔记
- 📦 **归类** — 把所有提到过的工具、网站、技能整理成清单

最终你得到的是一个 Obsidian 可以直接打开的笔记本，每节课都有独立的笔记页面，所有工具和技能都列好了表格。

## 第一次使用

### 打开知识库
用 Obsidian 打开项目文件夹，你会看到这些页面：

```
🏠 知识库总览.md     ← 入口首页（从这开始）
📋 知识库条目.md     ← 所有知识点汇总
📚 课程总结/         ← 每节课的独立笔记（都在这里）
📦 资产库/           ← 工具/网站/技能清单

还有原始课件放在 raw-slides/ 和 raw-transcripts/ 里
```

### 运行一次完整处理
第一次使用时，需要让系统读取所有课件并生成笔记：

1. 在项目文件夹里，找到 **`run_pipeline_with_key.sh`** 这个文件
2. 双击它
3. 等几分钟就好

处理完成后，刷新 Obsidian，你就能看到所有课程笔记了。

> 💡 如果双击后弹出一个黑色窗口闪了一下就消失了，可以试试右键 → 用终端打开

## 每次学完新课后

### 第一步：放课件
把老师发的课件（.md 文件）拖进 **`raw-slides/`** 文件夹。

### 第二步：放转写
如果有录音转写的文本（.txt 文件），拖进 **`raw-transcripts/`** 文件夹。

### 第三步：更新知识库
双击 `run_pipeline_with_key.sh`，等它处理完。

### 第四步：在 Obsidian 里看
刷新 Obsidian，你会看到：
- **📚 课程总结/** 里多了一个新的课程笔记
- **📦 资产库/** 里的工具和技能清单也自动更新了

> 💡 **每次上完课就这三步**：放文件 → 双击脚本 → 在 Obsidian 看结果。

## 在 Obsidian 里怎么看

### 看所有课程一览
打开 **🏠 知识库总览.md**，这里是整个知识库的入口。

- 上半部分列了所有课程的链接，点一下就能跳到对应笔记
- 下半部分有工具和技能的快捷入口

### 复习某节课
在 **📚 课程总结/** 文件夹里找到那节课，直接打开。每个笔记都包含：

| 内容 | 说明 |
|------|------|
| 📝 课程概要 | 这节课讲了什么 |
| 🎯 核心知识点 | 知识点拆解 |
| 💡 关键 takeaways | 最重要的结论（必须记住的） |
| 🛠 工具与软件 | 这节课提到的所有工具 |
| 💪 学到技能 | 这节课学到的能力 |
| 📎 推荐资源 | 老师同学推荐的网站和工具 |

### 找一个工具
打开 **📦 资产库/ 🛠 工具清单.md**，所有工具按出现频率排列。
可以看到每个工具在哪些课程里出现过。

### 找一个技能
打开 **📦 资产库/ 💡 技能清单.md**，所有技能同样按频率排序。

### 找网站
打开 **📦 资产库/ 🌐 网站清单.md**，所有被推荐或引用过的网站都在这里。

## 文件夹里面都是什么

```
项目文件夹/                       ← 用 Obsidian 打开这个文件夹
│
├── 🏠 知识库总览.md               ← 首页（所有链接的入口）
├── 📋 知识库条目.md               ← 所有知识点的大汇总
├── KNOWLEDGE_BASE_MANUAL.md       ← 你现在看的这个手册
│
├── 📚 课程总结/                   ← 每节课的笔记都在这里
├── 📦 资产库/                     ← 工具/网站/技能清单
│   ├── 🛠 工具清单.md              ← 按频率排序的工具表格
│   ├── 🌐 网站清单.md              ← 所有引用过的网址
│   └── 💡 技能清单.md              ← 所有学到的技能
│
├── raw-slides/                    ← 📥 放课件的地方（手动放入）
├── raw-transcripts/               ← 📥 放录音转写的地方（手动放入）
├── raw-feishu/                    ← 📥 放飞书群聊的地方（可选）
│
├── processed/                     ← 系统自动整理后的中间文件
├── knowledge_base/                ← 系统自动生成的知识点
├── assets/                        ← 系统自动生成的资产数据
│
├── pipeline/                      ← 系统内部用的程序代码
│   ├── config.py                  ← 配置文件（可自定义）
│   ├── ingestion.py               ← 数据导入
│   ├── cleaning.py                ← 数据清洗
│   ├── llm_client.py              ← AI 调用（带重试）
│   ├── knowledge_extraction.py    ← 知识提取
│   ├── course_summarization.py    ← 课程总结
│   ├── asset_extraction.py        ← 资产提取
│   ├── export_markdown.py         ← Markdown 导出
│   └── feishu_export_guide.md     ← 飞书导出指南
│
├── run_pipeline.py                ← 🚀 处理流程入口
├── run_pipeline_with_key.sh       ← 🔑 双击这个就能运行
├── README.md                      ← 项目介绍
├── LICENSE                        ← 开源协议
└── .gitignore                     ← Git 忽略规则
```

你平时只需要关心这些文件夹：

| 文件夹 | 你要做什么 |
|--------|-----------|
| `raw-slides/` | 课后把课件拖进来 |
| `raw-transcripts/` | 课后把转写拖进来 |
| `📚 课程总结/` | 看笔记 |
| `📦 资产库/` | 找工具和技能 |
| `🏠 知识库总览.md` | 入口页 |

其他文件夹都是系统自己用的，你不需要动它们。

## 常见问题

### 双击脚本后没有反应？
试试右键点击 `run_pipeline_with_key.sh` → 用「终端」打开。
第一次运行可能会弹出一个权限询问窗口，点「允许」就行。

### API Key 填在哪？
打开 `run_pipeline_with_key.sh`，把文件里 `YOUR_API_KEY_HERE` 替换成你的真实 Key。

支持的 API：
- **OpenAI** — `https://api.openai.com/v1`，模型 `gpt-4o-mini`
- **DeepSeek** — `https://api.deepseek.com`，模型 `deepseek-chat`
- 任何兼容 OpenAI 格式的 API 都可以

### 想重新生成所有笔记？
双击 `run_pipeline_with_key.sh` 就行。
它会自动跳过已经做过的步骤，只重新生成分析和总结。

### 想换一个 AI 模型？
修改 `run_pipeline_with_key.sh` 里的三行配置：

```
OPENAI_API_KEY="你的 Key"
OPENAI_BASE_URL="https://api.xxx.com/v1"
PIPELINE_MODEL="模型名称"
```

### 飞书群聊内容怎么加进来？
如果机器人没有加入群聊的权限，可以用飞书桌面版手动导出聊天记录：

1. 打开飞书 → 进入群聊 → 右上角菜单 → 导出聊天记录
2. 把导出的文件放进 `raw-feishu/` 文件夹
3. 双击 `run_pipeline_with_key.sh` 更新

### 生成的 takeaways 不够精炼？
可以试试换更强的模型，比如 GPT-4 或 Claude Sonnet。
修改 `run_pipeline_with_key.sh` 里的 API 配置即可。

### 文件中有乱码或奇怪符号？
系统会自动清理大部分格式问题。如果某个课件显示有问题，可以检查一下原始 .md 文件是否包含特殊格式。

### 想添加之前学过的旧课程？
和添加新课程一样：把课件放进 `raw-slides/` → 把转写放进 `raw-transcripts/` → 双击脚本 → 刷新 Obsidian
